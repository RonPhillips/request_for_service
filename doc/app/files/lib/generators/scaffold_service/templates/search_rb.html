Adds search method to ActiveRecord::Base. The query language supports the operators (), not, and, or Precedence in that order.

  • is an alias for not.

If no operator is present, the operator "and" is assumed. Lastly, anything within double quotes is treated as a single search term.

For example, 

 ruby rails => records where both ruby and rails appear
 "ruby on rails" => records where "ruby on rails" appears
 ruby or rails => records where ruby or rails (or both) appears
 ruby or chunky bacon => records where ruby appears or
                         both chunky and bacon appear
 not dead or alive => records where alive appears or dead is absent
 -(ruby or rails) => records where neither ruby nor rails appears
 (ruby or rails) -"ruby on rails" => records where ruby or rails appears
                                     but not the phrase "ruby on rails"
